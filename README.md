### Модуль 1 - Основы Go + Git

#### Игра "Угадай число" на Go

Простая консольная игра, в которой игроку нужно угадать случайное число от 1 до 100 за ограниченное количество попыток.

- При запуске игра генерирует случайное число в зависимости от сложности.
- У игрока есть N попыток, чтобы угадать число.
- После каждой попытки программа сообщает:
    - Если число меньше загаданного — "Секретное число больше👆"
    - Если число больше загаданного — "Секретное число меньше👇"
    - Если число угадано — сообщение о победе.
- Если попытки закончились, игрок проигрывает, и показывается правильный ответ.

В игре также присутствует

1. **Проверка ввода**
    - Обработка ошибок при вводе (если пользователь ввёл текст вместо числа).
2. **Подсказки по расстоянию**
    - Добавить уровни подсказок:
        - Разница ≤5: 🔥 "Горячо"
        - Разница ≤15: 🙂 "Тепло"
        - Иначе: ❄️ "Холодно"
3. **Запоминание прошлых попыток**
    - Выводить список введённых ранее чисел.
4. **Выбор сложности**
    - Easy: 1–50, 15 попыток
    - Medium: 1–100, 10 попыток
    - Hard: 1–200, 5 попыток
5. **Сыграть ещё раз**
    - После окончания игры предложить сыграть повторно без перезапуска программы.
6. **Сохранение лучших результатов**
    - Ведение файла `results.json`, где сохраняется дата, исход (победа/проигрыш), количество попыток.

### Модуль 2 - Конкурентность в Go
Проект представляет собой систему денежных переводов между пользователями.

Проект данного модуля состоит из 4 частей. Функционал каждой части будет описан ниже

#### Часть 1. Основы
В рамках первой части выполнено следующее:
1. Создание структуры ```User```, который имеет идентификатор, имя и баланс
2. Реализация метода ```Deposit``` по добавлению средств на баланс пользователя
3. Реализация метода ```Withdraw``` по списанию средств с баланса пользователя
4. Реализация симуляции по заведению пользователей и симуляция работы с их балансами

#### Часть 2. Добавление транзакций и их обработка
В рамках второй части выполнено следующее:
1. Создание структуры ```Transaction```, которая имеет идентификатор пользователя, у которого будет списание средств, идентификатор пользователя, которому поступят средства, сумма списания/зачисления
2. Реализация структуры ```PaymentSystem``` которая содержит в себе мапу пользователей и список транзакций
3. Реализация метода ```ProcessingTransactions``` который обрабатывает все транзакции PaymentSystem
4. Реализация симуляции по заведению пользователей, транзаций и симуляция работы PaymentSystem

#### Часть 3. Конкурентная обработка транзакций
В рамках третьей части выполнено следующее:
1. Создание нескольких паралелльных workers, которые выполняют применения транзакций. Каждый worker добавляется в Wait группу
2. Реализация метода ```Worker``` которая обрабаывает транзакции из канала с транзакциями
3. Метод```ProcessingTransactions``` теперь обрабатывает переданную ему транзакцию
4. Реализация симуляции по заведению пользователей, транзаций и симуляция работы PaymentSystem при нескольких параллельных worker`ах

#### Часть 3. Конкурентный доступ к балансу пользователей
В рамках четвертой части выполнено следующее:
1. Реализация конкурентного доступа к балансу пользователей
2. Реализация симуляции по заведению пользователей, транзаций и симуляция работы PaymentSystem при нескольких параллельных worker`ах
